# Profile Context

**Persona**: Frontend Developer and Freelance Designer

## Persona & Background

- **Professional background**: Experienced UI developer building agency-grade websites.
- **Project role**: Solo creator, designer, and developer.
- **Industry context**: Freelance web design and development.

## Technical Expertise

- **Strong areas**: HTML semantics, CSS layout (Grid/Flexbox), basic JavaScript interactivity.
- **Weak areas**: Complex build tools (Webpack, Vite), CI/CD pipelines, server management.
- **Learning goals**: Improving Lighthouse scores and learning basic Github Actions setup.

## Communication Preferences

- **Decision making**: Present pros/cons if design choices affect performance.
- **Tone & voice**: Direct and practical. Short and to the point.
- **Answer format**: Code snippets favored over long conceptual essays.

## Development Philosophy

- **Trade-offs**: Readability and simplicity over cleverness.
- **Core principles**: Keep the build process invisible or non-existent.
- **Risk tolerance**: High. Small bugs on a static site are acceptable if they don't break the layout.

## AI Interaction Rules

- **Autonomy**: Output full, working HTML/CSS/JS logic. Do not leave "TODO" comments.
- **Verification**: Ensure the site looks good on mobile and desktop viewports in your mind before proposing HTML.
- **Focus**: I am visual. If you change the UI, explain _why_ it looks better. Ask for confirmation before adding any npm packages.
