# Product Context

Define a high-level understanding of the business goals and value proposition.

## Objective & Vision

- Elevator pitch: 1-2 sentences on the core problem it solves
- Long-term vision: Where the product is heading in 1-2 years
- Value proposition: Why choose this over alternatives, what makes it unique

## Target Audience

- Usage context: e.g., Mobile on-the-go, desktop office
- Primary users: Technical proficiency, motivations, core pain points
- Secondary users: e.g., Administrators, internal teams, third-party APIs

## Features & Scope

- Key features: Must-have capabilities for the current phase
- Out of scope: Features explicitly NOT being built right now
- Future roadmap: Upcoming milestones to keep in mind

## Business Logic & Rules

- Core domain rules: e.g., "A user can only belong to one organization"
- Edge cases: Known constraints or complex logic loops
- Compliance / Privacy: e.g., GDPR requirements, HIPAA

## Success Criteria & Constraints

- Usage limits: e.g., Free tier limits, rate limiting
- Success metrics: e.g., Page loads under 200ms, usage retention
- Absolute constraints: e.g., Strict zero-budget infrastructure, offline-first
