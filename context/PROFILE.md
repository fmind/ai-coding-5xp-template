# Profile Context

Define yourself to help the AI tailor its communication style, tone, and technical depth.

## Persona & Background

- Professional background: e.g., Senior Backend Engineer, Product Manager
- Project role: e.g., Solo founder, tech lead, frontend specialist
- Industry context: e.g., FinTech, EdTech, Open Source

## Technical Expertise

- Strong areas: Technologies where the AI can skip basic explanations
- Weak areas: e.g., CSS styling, CI/CD, where complete code is needed
- Learning goals: Skills you actively want to improve during the project

## Communication Preferences

- Decision making: e.g., Make assumptions or present pros/cons list
- Tone & voice: e.g., Direct and brief, academic, casual and encouraging
- Answer format: e.g., Code snippets first versus step-by-step explanations

## Development Philosophy

- Trade-offs: e.g., Speed over optimization, readability over cleverness
- Core principles: e.g., "Done is better than perfect", "DRY at all costs"
- Risk tolerance: e.g., "Move fast and break things", or "High reliability first"

## AI Interaction Rules

- Autonomy: e.g., "Propose entire files" versus "Show diffs only"
- Verification: e.g., "Remind me to run tests after big changes"
- Focus: e.g., "Only answer what I ask, no unsolicited advice"
